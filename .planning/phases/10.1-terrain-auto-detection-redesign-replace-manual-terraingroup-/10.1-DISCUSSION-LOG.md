## Decisions Captured

### Scope Boundary — What Survives vs Gets Deleted
**Q: What from Phase 10 survives Phase 10.1 vs gets deleted?**
→ Keep variation modes + source_id + slope (Recommended)

**Q: Phase 10.1 deletes PentaTileTerrainGroup + terrain_group export. Fallback routing goes back to... what?**
→ Back to layout fallback (Recommended) — v0.2.0 behavior: tile_set == null routes to layout.get_fallback_tile_set()

**Q: _auto_detect_terrains fires on which signals?**
→ tile_set.changed + setter (both). NOT in _ready() (setter suffices for scene deserialization).

**Q: How should the simplified dual-grid dispatch work?**
→ Per-corner, single layout (Recommended). Each corner dispatches through shared layout with strip_index=terrain_id. sample_fn filters same-terrain neighbors as filled.

**Q: Anything else from Phase 10 to delete beyond TerrainGroup?**
→ TerrainGroup only (Recommended). Keep source_id, terrain_mode(), compute_mask(strip_index), variation_mode, Slope, passthrough.

### Test Strategy
**Q: What new tests should ship?**
→ 4 tests: terrain_autodetect_test, terrain_dispatch_test, terrain_determinism_test, terrain_sample_terrains_test. Delete 5 old TerrainGroup-coupled tests.

**Q: What pattern set for terrain dispatch testing?**
→ 8 key patterns (Recommended): L_shape, T_shape, plus_shape, 2×2_bicolor, line_h_int, hollow_ring, 1×1, 3×3.

**Q: New filenames or rewrite in-place?**
→ New filenames (Recommended). Clean break from old TerrainGroup-coupled test files.

### Dual-Grid Terrain Boundary Dispatch
**Q: How do dual-grid corners know which neighbors count as 'filled'?**
→ Same-terrain filtering in sample_fn (Recommended). Each corner's compute_mask only counts same-terrain neighbors as filled.

**Q: What does terrain_mode() actually DO in Phase 10.1?**
→ Report mode for terrain_set creation (Recommended). Used to set Godot TerrainMode when creating terrain_set 0. Not used for solving.

**Q: How does user configure terrain names/colors without TerrainGroup?**
→ Godot native TerrainSets inspector (Recommended). Auto-detection creates "Terrain N" defaults; user customizes in TileSet bottom panel; snapshot-before-clear preserves customizations.

### Variation (Preserved from Phase 10)
**Q: How are variation pools indexed without TerrainGroup?**
→ Per-terrain pools via terrain index (Recommended). mask_to_atlas scans atlas row for tiles sharing mask config.

**Q: Does Phase 10.1 preserve the hash shape?**
→ Preserve Phase 10 hash (Recommended): hash(Vector4i(cell.x, cell.y, terrain_id, _global_variation_seed)).

**Q: Does STRIP mode stay or get redesigned?**
→ Keep STRIP per terrain row (Recommended). STRIP picks from atlas row = terrain_id.

### Auto-Detect UX
**Q: Does auto-detection fire in _ready()?**
→ No — setter suffices. tile_set setter fires during scene deserialization.

**Q: How does _resolve_terrain_id map atlas_coords to sequential terrain ID across sources?**
→ Use TileData.terrain directly (Recommended). TileData.terrain is set to sequential ID during auto-detection.

**Q: Which Godot TerrainMode for terrain_set 0?**
→ MATCH_CORNERS default (Recommended). Covers 6/9 layouts. Wang2Edge/Min3x3 users manually adjust.

**Q: Terrains for all rows or only populated rows?**
→ All rows — artist may add tiles to empty rows later.

## Deferred Ideas
- Per-terrain Penta synthesis bridge — deleted with TerrainGroup; single shared layout makes this unnecessary
- Editor "Autodetect Terrains" button — rejected; fully automatic preferred
- Variation redesign — variation survives Phase 10.1 unchanged from Phase 10
